# d07_subset_manifest

- Description: Record which node subsets are considered valid candidates and why.

- Physical topology used for subset interpretation: Node A ESP32-C5 at `12 o'clock` / `0 degrees`, router/AP at `6 o'clock` / `180 degrees`, Node B RPi5 at `9 o'clock` / `270 degrees`, Node B RPi4 with USB-connected ESP32-C5 at `3 o'clock` / `90 degrees`, Node C RPi5 at `7:30` / `225 degrees` on the `1.00 m` ring, and Node C ESP32-S3 power-only receiver at the off-ring `1:30` / `45 degrees` corner, `2.00 m` from the experiment center.

- Subset {A,B}: valid D07 candidate. Node A used the live `primary_ap` ESP32-C5 stream captured through the primary-router UDP DNAT listener; Node B used the live `rpi5_ap_b` ESP32-C5 AP-B UDP summary stream. Final QC counted `8576` CSI rows for Node A and `7515` CSI rows for Node B, with `max_dropped=0`, `max_zero_len=0`, and `max_timeouts=0` on both streams. Cooperative timing for `esp32-c5-a -> esp32-c5-b` used 15 common pulses and was `trustworthy` with max bracketing gap `6.000 ms`, max uncertainty `5.500 ms`, and max residual `6.714 ms`.
- Subset {A,C}: valid D07 candidate. Node C used the live `rpi5_node_c_24g` ESP32-S3 stream on the dedicated Node-C 2.4 GHz AP path, captured from `<node-c-ap-address>:<ssh-port>` over the laptop TL-WN722N adapter. Final QC counted `8576` CSI rows for Node A and `18702` CSI rows for Node C, with `max_dropped=0`, `max_zero_len=0`, and `max_timeouts=0` on both streams. Cooperative timing for `esp32-c5-a -> esp32-s3-c` used 15 common pulses and was `trustworthy` with max bracketing gap `6.000 ms`, max uncertainty `5.500 ms`, and max residual `5.011 ms`.
- Subset {A,B,C}: valid D07 full heterogeneity candidate. All three streams were live, synchronized on the same operator-guided timeline, and completed all planned blocks `W0, E1, S1, M1, E2, S2, M2, E3`. Final live sensor row ratio was `2.49`, below the configured `4.0` gate; all three sensors reported `max_dropped=0`, `max_zero_len=0`, and `max_timeouts=0`; cooperative timing was `trustworthy` across both auxiliary pairs. Use this subset as the primary D07 claim surface, with the runbook topology and Node-C physical placement notes preserved for interpretation.
